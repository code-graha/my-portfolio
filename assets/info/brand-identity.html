04 ‚Äî Design Tokens

Spacing & Border Tokens

Consistent spacing and border-radius tokens keep the layout harmonious across all components.

üí°
In plain English

Tokens are named values that store design decisions so the same number is used everywhere. Instead of writing margin: 16px everywhere, a token like --sp-4 ensures all 16px gaps stay consistent. The spacing scale controls gaps between elements; border-radius controls how rounded corners look; shadows control depth and the glowing effect on hover.

07 ‚Äî Brand Voice & Identity

Personality & Guidelines

The brand communicates as a senior engineer ‚Äî precise, confident, and approachable. It avoids jargon overload while demonstrating deep technical fluency.

üí°
In plain English

Brand voice is how the portfolio "sounds" when you read it ‚Äî the words chosen, the tone of headings, and how achievements are described. Think: a senior engineer explaining work to a smart colleague, not a salesperson pitching to a client. Be specific about technologies, lead with outcomes, and stay warm without being casual. The Do / Don't list below shows real examples of correct vs incorrect wording.

‚öôÔ∏è

Precise

Every word earns its place. Technical details are accurate and specific ‚Äî "Kubernetes + Istio" not just "containerization".

üéØ

Confident

Statements are direct. No excessive hedging. Results and experience are stated clearly with supporting evidence.

ü§ù

Approachable

Technical depth doesn't mean cold. The tone is warm and collaborative ‚Äî "Let's build something amazing together."

üî≠

Forward-thinking

Cloud-native, modern DevOps, emerging technologies ‚Äî the brand positions at the cutting edge, not the trailing edge.

üßë‚Äçüíª

Engineer-first

The terminal aesthetic, mono type, and code snippets reinforce that this is an engineer speaking, not a marketer.

üìê

Minimal

No decoration for its own sake. Every visual element has a purpose ‚Äî glow orbs hint at depth, grid lines hint at structure.

‚úì Do

  • Use technical terms accurately and specifically
  • Lead with outcomes ("cut root cause analysis time by 50%")
  • Use the terminal / code motif consistently
  • Pair neon cyan with dark backgrounds exclusively
  • Keep section labels in mono uppercase
  • Use gradients on primary CTAs and hero text
  • Always animate with purpose ‚Äî guide, not distract
  • Respect reduced-motion preferences

‚úï Don't

  • Use generic buzzwords without context
  • Mix more than 3 accent colors in one component
  • Place the logo on low-contrast backgrounds
  • Use light backgrounds as primary surfaces
  • Override Inter with another sans-serif
  • Use rounded sans-serifs for code blocks
  • Add glow effects to non-interactive, flat elements
  • Use gradients for body text ‚Äî only headings/CTAs
